# 제품 요구사항 정의서 (Product Requirements Definition)

## 1. 개요 및 논의 사항

### 추가 논의 필요 사항
*   **별점 or 후기 기능**: 도입 여부 및 방식 결정 필요.
*   **AI 활용**: 상품 설명 자동 생성, 적정 시세 추천 등 도입 검토.
*   **결제 기능**: PG사 연동 vs 내부 포인트 시스템 결정 필요.

## 2. 기능 요구사항 (Functional Requirements)

| 구분 | 요구사항 ID | 요구사항 명 | 요구사항 상세 설명 | 비고 |
| :--- | :--- | :--- | :--- | :--- |
| **회원가입** | REQ-001 | 회원가입 | 사용자는 회원가입을 할 수 있어야 한다 | |
| **로그인** | REQ-002 | 로그인(일반 회원) | 사용자는 로그인 할 수 있어야 한다 | |
| | REQ-003 | 로그인(소셜 회원) | 소셜 회원의 경우 [구글], [네이버], [카카오] 3가지 소셜 수단으로 로그인 할 수 있어야 한다 | |
| | REQ-004 | 아이디 찾기(일반 회원) | 회원은 본인의 성명, 핸드폰 번호, 이메일 주소를 입력하고 일치할 경우 아이디 정보를 제공한다 | |
| | REQ-005 | 비밀번호 재설정(일반 회원) | 회원 본인 아이디, 이메일 주소를 입력하고 일치할 경우 비밀번호 재설정 인증 메일을 발송한다. 회원은 인증 메일을 통해 비밀번호를 재설정 한다 | |
| **회원정보** | REQ-006 | 회원 정보 조회 | 회원은 본인의 회원 정보를 조회할 수 있어야 한다 | |
| | REQ-007 | 회원 정보 수정 | 아이디, 이름, 생년월일, 이메일 주소 변경 불가. 회원은 그 외의 정보를 수정 할 수 있어야 한다 | |
| | REQ-008 | 회원 탈퇴 | 회원은 언제든지 서비스를 탈퇴할 수 있어야 한다. 탈퇴 시 진행 중인 경매나 거래가 없어야 하며, 법적 보관 의무가 없는 데이터는 삭제 처리한다. | 진행 중인 거래 확인 필수 |
| **상품 관리** | REQ-101 | 상품 등록 | 회원은 상품을 등록할 수 있어야 한다 | |
| | REQ-102 | 상품 정보 | 상품에는 제목, 설명, 이미지, 시작가, 경매 종료 시간, 즉시구매가(선택사항)이 포함되어야 한다 | |
| | REQ-103 | 상품 목록 조회 | 사용자는 상품 목록을 조회할 수 있어야 한다 | |
| | REQ-104 | 상품 상세 조회 | 사용자는 상품 상세 정보를 조회할 수 있어야 한다 | |
| | REQ-105 | 끌어올리기 기능 | 판매자는 본인이 등록한 상품을 목록 최상단으로 재노출시킬 수 있다. | 횟수 제한(예: 1일 1회) |
| | REQ-106 | 정렬 기준 업데이트 | '끌어올리기' 수행 시 해당 상품의 정렬용 기준 시간 데이터가 현재 시간으로 갱신된다. | |
| **경매 및 입찰** | REQ-201 | 입찰 | 회원은 경매 상품에 입찰할 수 있어야 한다 | |
| | REQ-202 | 입찰 금액 검증 | 입찰 금액은 현재 최고 입찰가보다 높아야 한다, 입찰 단위별로 경매 금액을 높일 수 있어야 한다 | |
| | REQ-203 | 경매 종료 | 경매는 설정된 종료 시간이 지나면 자동 종료되어야 한다 | |
| | REQ-204 | 낙찰자 선정 | 경매 종료 시 최고 입찰자를 낙찰자로 지정해야 한다 | |
| | REQ-205 | 자동 연장 | 종료 직전 입찰 시 경매 종료 시간이 연장될 수 있어야 한다 | |
| | REQ-206 | 입찰 내역 조회 | 회원은 자신이 입찰한 내역을 조회할 수 있어야 한다 | |
| **구매/소통** | REQ-301 | 즉시구매가 설정 | 상품에 즉시구매 가격을 설정할 수 있어야 한다 | |
| | REQ-302 | 즉시구매 처리 | 즉시구매가 발생하면 경매는 즉시 종료되어야 한다 | |
| | REQ-303 | 댓글 작성 | 회원은 상품 상세 페이지에서 문의 또는 의견을 위해 댓글을 작성할 수 있다. | |
| | REQ-304 | 답글 작성 | 회원은 특정 댓글에 대해 답글을 작성하여 계층형 대화 구조를 형성할 수 있다. | 판매자 답변 등 |
| | REQ-305 | 댓글/답글 삭제 | 사용자는 본인이 작성한 댓글이나 답글을 삭제할 수 있다. | 삭제 시 '삭제된 댓글입니다' 표시 |
| | REQ-306 | 비밀 댓글 설정 | 댓글 작성 시 판매자와 작성자만 볼 수 있도록 비밀글 설정을 할 수 있다. | |
| | REQ-307 | 댓글 알림 | 내 상품에 댓글이 달리거나 내가 쓴 댓글에 답글이 달리면 알림을 발송한다. | |
| **거래 관리** | REQ-401 | 거래 생성 | 낙찰 또는 즉시구매 시 거래가 생성되어야 한다 | |
| | REQ-402 | 거래 상태 관리 | 거래는 상태값(진행중, 거래완료, 취소 등)을 가져야 한다 | |
| | REQ-403 | 내 거래 목록 조회 | 회원은 자신의 거래 목록을 조회할 수 있어야 한다 | |
| **검색/시세정보** | REQ-501 | 상품 검색 | 사용자는 상품을 검색할 수 있어야 한다 | |
| | REQ-502 | 판매자 검색 | 사용자는 판매자의 닉네임을 검색할 수 있어야 한다 | |
| | REQ-503 | 검색 결과 조회 | 검색 결과는 상품 목록 형태로 조회할 수 있어야 한다 | |
| | REQ-504 | 정렬 조건 | 검색 결과는 가격, 종료 시간 기준으로 정렬할 수 있어야 한다 | |
| | REQ-505 | 필터 조건 | 검색 결과는 진행 상태(진행중/종료)로 필터링할 수 있어야 한다 | |
| | REQ-506 | 모델별 시세 그래프 | 특정 모델(상품)의 과거 낙찰 기록을 기반으로 기간별 시세 추이를 그래프로 제공한다. | |
| | REQ-507 | 가격 등락폭 표시 | 동일 모델의 직전 낙찰가 대비 현재가(또는 최근 거래가)의 등락 금액과 퍼센트(%)를 표시한다. | |
| **관리자 기능** | REQ-601 | 상품 관리 | 관리자는 상품을 수정 또는 삭제할 수 있어야 한다 | |
| | REQ-602 | 신고 처리 | 관리자는 신고된 상품 혹은 회원을 검토할 수 있어야 한다 | |
| | REQ-603 | 대시보드 조회 | 관리자는 전체 회원 수, 일별 경매 등록 수, 낙찰률 등 서비스 지표를 조회할 수 있어야 한다. | |
| | REQ-604 | 회원 상태 관리 | 관리자는 불량 이용자(허위 입찰, 욕설 등)에 대해 이용 정지 또는 강제 탈퇴 처리를 할 수 있어야 한다. | |
| | REQ-605 | 입찰 취소 권한 | 명백한 오류나 부정행위가 발견된 경우, 관리자는 특정 입찰 건을 강제로 취소할 수 있어야 한다. | |
| | REQ-606 | 카테고리 관리 | 관리자는 상품의 카테고리(전자제품, 의류 등)를 추가하거나 관리할 수 있어야 한다. | |
