# 업무 분장 및 산출물 현황

## 1. 초기 기획 및 정의 단계
*   프로젝트 기획서 (Project Proposal/Plan) → @이
*   요구사항 정의서 (SRS) , PRD → @강
*   와이어프레임 및 스토리보드 → @윤 @천
    *   UI (v0) → @천, @이 (진행중)

## 2. 설계 및 개발 준비 단계
*   기술 스택 결정 및 선정 이유서 → @천
*   시스템 아키텍처 명세서 → @윤 @강 @이
*   데이터베이스 명세서 (ERD 포함) → @윤
*   API 명세서 (OpenAPI/Swagger) → @강 + 개발팀(@윤, @천)

## 3. 개발 및 테스트 단계
*   주간/일일 회고 및 진행 보고서 (Scrum Retrospective)
*   테스트 케이스 및 결과서
*   Git Commit Log 및 컨벤션

## 4. 최종 정리 및 유지보수 단계
*   최종 프로젝트 보고서
*   배포 및 운영 매뉴얼
*   개인 회고록
